# 2026-03-30

## Durable implementation facts

- Dashboard state is now composed from independently cached slices instead of a single full recompute path.
- The dashboard hot path uses quick memory status; explicit semantic-memory prepare runs as a persisted background operation with websocket-visible state.
- Job output and terminal transcripts use batched, byte-bounded UTF-8-safe buffering on both server and web paths.
- The hot client routes now keep markdown lazy, prefetch only `Files` and `Settings` after auth, and share viewport measurement logic across Chat and Terminal.
- Perf artifacts and budgets now track cold dashboard, visible first token, memory prepare accepted/completion, and bundle size, and `pnpm perf:check` enforces those budgets against the current baseline.
- Decision handling is canonically normalized through `decision-service`, `packages/shared/src/decisions.ts`, and `apps/server/src/lib/decision-updates.ts`; legacy approval, memory-draft, and pairing surfaces are compatibility views over the same ledger.
- Repo hygiene now has a first-class guardrail: `pnpm hygiene:check` validates canonical helper ownership, production file-size limits, required docs, and script reachability, and `pnpm verify` includes it.
- The canonical surface inventory lives in `docs/surface-inventory.md`, and OpenClaw boundary guidance is explicitly linked from `docs/openclaw-boundary.md` and architecture docs.
