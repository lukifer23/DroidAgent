# Personal Preferences

Use this file for stable operator preferences that should make DroidAgent feel more personal and more useful over time.

## Tone and output

- terse, direct, high-signal replies
- summarize first and keep explanations practical
- no fluff, no placeholder copy, no fake progress

## Engineering defaults

- prefer real, production-grade implementations over mock flows
- avoid stubs, placeholders, silent failures, and duplicate controls
- keep changes hardened, streamlined, and easy to operate from the phone

## Product priorities

- mobile-first UI that scales cleanly to foldable screens
- low-latency live chat with real OpenClaw tooling and clear feedback
- local-first runtimes, local semantic memory, and fast setup
- one-tap or near-one-tap operator flows whenever possible

## Workflow

- treat the current repository as the active workspace by default
- prefer local runtimes before cloud providers unless explicitly changed
- keep commands non-interactive and easy to rerun
- surface the actual live model, tools, memory, and permissions state in the UI

## Do not put here

- API keys
- passwords
- temporary task notes
